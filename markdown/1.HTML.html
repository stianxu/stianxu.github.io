DOCTYPE 文档类型 #

DOCTYPE

  • DOCTYPE 即 Document Type，网页文件的文档类型。
  • DOCTYPE 目的是告诉浏览器应该以什么样的文档类型来解析文档。如：<!DOCTYPE html>
  • DOCTYPE 必须声明在 HTML ⽂档的第⼀⾏。

head 标签 #

head 标签作用

  • head 标签用于定义文档的头部，它是所有头部元素的容器。
  • head 标签中可以引用脚本、样式表，描述页面信息，书写内部样式，做移动端适配操作。

head 标签不可缺少部分

  • 下面这些标签可用在 head 部分： meta、link、 script、 style、 title。

  • 其中 title 定义文档的标题，它是 head 部分中唯一必需的标签。

meta 标签 #

meta 标签

  • <meta> 标签的功能由 name/http-equiv 和 content 两部分属性来定义。

    • 如果设置 name 属性，可以设定移动端适配，或描述网页文档的信息（例如：编码类型、⽹⻚描述、 关键词等）。

    • 如果设置 http-equiv 属性，可以向浏览器传递信息（例如：设定网页到期时间，网页缓存等）。

一些常用的功能及写法

  • charset，用来描述 HTML 文档的编码类型

    • html
      <meta charset="UTF-8" >

  • keywords，页面关键词（SEO)

    • html
      <meta name="keywords" content="关键词" />

  • description，页面描述（SEO)

    • html
      <meta name="description" content="页面描述内容" />

  • viewport，适配移动端，可以控制视口的大小和比例

    • html
      <meta name="viewport" content="width=device-width, initial-scale=1.0">

  • expires，期限，可以用于设定网页的到期时间。一旦网页过期，必须到服务器上重新传输

    • html
      <meta http-equiv="expires" content="Wed, 20 Jun 2007 22:33:00 GMT">

  • cache-control，缓存控制，设置再访问这个网站要重新下载

    • html
      <meta http-equiv="cache-control" content="no-cache">

label 作用

<input type="text" id="mobile"/>

title 与 h1、b 与 strong、i 与 em 标签 #

title 与 h1 的区别

  • title 只表示是个标题，H1 则表示层次明确的标题，对页面信息的抓取有很大的影响。

b 与 strong 的区别

  • b 标签只是一个简单加粗标签，strong 标签另有语义，起到加重语气的效果，搜索引擎更侧重 strong 标签。

i 与 em 的区别

  • i 只是将内容展示为斜体，em 则表示强调的文本。

行内标签

块级标签

空标签，即没有内容的 HTML 标签，空标签没有闭合标签

src 属性 和 href 属性 #

src 的特性

  • 引用外部资源

    • 比如script标签、img标签、video标签。

  • 会替换标签的内容

    • 在 script 引入标签中书写内容，会被引入的 script 替换

  • 会暂停其他资源的下载

    • 当浏览器解析到使用 src 的元素时，会暂停其他资源资源的下载，直到 src 引用资源加载、编译、执行完毕，这也是为什么 script 元素推荐放在 html 结构的底部。

href 的特性

  • 表示超链接

    • 比如a标签、link标签，表示外部资源与该页面的联系。

  • 不会替换元素本身的内容

    • 在 a 标签中添加图片，点击 a 标签跳转页面，但是图片内容没有被替换

  • 不会暂停其他资源的下载

    • 像 CSS 那样影响页面观感的可以放在 html 结构的头部优先加载。

核心思想上的区别

  • src 代表的是网站的一部分，没有会对网站的使用造成影响。
  • href 代表网站的附属资源，没有不会对网站的核心逻辑和结构造成影响。

script 标签中的 defer 和 async 属性 #

JS 脚本放置位置

  • 将 JS 脚本放在文档的底部，来使 JS 脚本尽可能的在最后加载执行，防止阻塞文档的解析。
  • 将 JS 脚本放在文档的头部，不添加 defer 或 async 属性，浏览器读到就开始 加载和执行 相应的脚本，此举会阻塞文档的解析。

defer 属性

  • 给 JS 脚本添加 defer 属性，defer 属性会让脚本的加载与文档的解析异步进行，然后在文档解析完成后再执行这个脚本文件，这样的话就能使页面的解析始终不被阻塞。
  • 多个设置了 defer 属性的脚本是顺序执行的。

async 属性

  • 给 JS 脚本添加 async 属性，async 属性会使脚本异步加载，不会阻塞文档的解析过程，但是当脚本加载完成后立即执行 JS 脚本，这个时候如果文档没有解析完成的话同样会造成阻塞。
  • 多个 async 属性的脚本的执行顺序是不可预测的。

data- 开头的自定义属性 #

自定义属性

  • 这是一种为 HTML 元素添加额外数据信息的方式，被称为 自定义属性。

自定义属性用途

  • DOM 操作，在元素标签上声明 data-num="123"，通过 dataset.num 操作

HTML、XML、XHTML 标记语言 #

都属于标记语言

语言中文名说明
HTML4超文本标记语言主要用于做界面呈现。HTML 是先有实现，后面才慢慢制定标准的，导致 HTML ⾮常混乱和松散，语法非常的不严谨。
XML可扩展标记语言主要⽤于存储数据和结构。语法严谨，可扩展性强。由于 JSON 也有类似作⽤但更轻量⾼效， XML 的市场变得越来越⼩。
XHTML可扩展超文本标记语言属于加强版 HTML，为解决 HTML 的混乱问题而生，在语法方面变得和 XML 一样严格。另外，XHTML 的出现也催生了 HTML5，让 HTML 向规范化严谨化过渡。
HTML5超文本标记语言在 HTML 的基础上进行拓展，用于页面呈现 (目前标准)。

HTML 语义化 #

语义化

  • 通俗来讲就是用合适的标签。

常见的语义化标签

  • header、nav、p、button、footer

语义化的优点

  • 对浏览器友好，更适合搜索引擎优化（SEO)。
  • 对开发者友好，语义类标签增强了可读性，便于开发与维护。

HTML 文档资源的解析规则 #

HTML

  • HTML 在接收到一部分之后就开始解析

CSS

  • CSS 下载和解析不会阻塞 DOM 的解析

JS

  • JS 的下载和解析都会阻塞 DOM 的解析
  • JS 的解析需要等待 CSSOM 全部解析完

页面渲染

  • 页面渲染需要 DOM 和 CSSOM 全部解析完
